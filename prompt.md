# 🌍 Hackathon Prompt – Project Alignment

## 📌 Prompt Summary
This hackathon focuses on building solutions that go beyond awareness and actively encourage **sustainable behavior change**.

Projects should not simply inform users, but instead help them take consistent, meaningful actions that contribute to sustainability.

---

## 🎯 Core Requirements

### 1. Behavior Change (Most Important)
Solutions must:
- Influence user behavior
- Encourage consistent action over time
- Move users from intention → action

❌ Not enough:
- Providing information only
- Raising awareness without follow-through

✅ Strong solutions:
- Guide users toward habits
- Reinforce consistency (e.g., reminders, feedback, rewards)

---

### 2. Clear Target User
Projects must define a **specific audience**, not “everyone.”

Examples of strong targeting:
- Urban apartment residents
- College students with limited time
- First-time plant owners

This ensures the solution is:
- Focused
- Realistic
- Relevant

---

### 3. Measurable Impact
Solutions should demonstrate impact, even if simplified.

Examples:
- Tracking user actions (e.g., completed tasks)
- Showing progress (e.g., streaks, scores)
- Estimating outcomes (e.g., improved plant health)

The goal is to show:
> “This product changes behavior in a measurable way.”

---

### 4. Feasibility (MVP Focus)
Projects should:
- Be achievable within the hackathon timeframe
- Prioritize a working prototype over complexity
- Avoid unnecessary technical overhead

❌ Avoid:
- Overly complex AI systems
- Features that cannot be demonstrated

✅ Focus on:
- Clear functionality
- Working interactions
- Demonstrable value

---

## 🌱 How Plantir Aligns

Plantir supports the prompt by:

### Behavior Change
- Encourages consistent plant care through reminders and guidance
- Reduces neglect and overwatering
- Builds sustainable habits over time

### Target User
- Urban, busy individuals with indoor plants
- First-time plant owners

### Measurable Impact
- Tracks plant care actions (e.g., watering)
- Shows plant health or status
- Can include streaks or progress indicators

### Feasibility
- Focuses on simple, effective features
- Avoids complex AI in MVP
- Prioritizes usability and clarity

---

## 🧠 Key Framing for Judges

When presenting, the project should be described as:

> A tool that helps users build sustainable habits through consistent plant care.

NOT:
> Just a plant care app

---

## 🧭 Guiding Principle

Always prioritize:
- Simplicity
- Clarity
- Demonstrable behavior change

If a feature does not clearly support behavior change or demo value, it should be deprioritized.

---

## 🚀 Success Criteria

The project succeeds if it can clearly show:
1. A defined user
2. A behavior being changed
3. A working feature that enables that change
4. A visible or measurable outcome

---

## ⚠️ Common Pitfalls to Avoid

- Being too broad (“for everyone”)
- Focusing on information instead of action
- Overengineering features that cannot be completed
- Lack of clear demo or impact

---

## 🌿 Final Takeaway

This hackathon is not about building the most complex product.

It is about building:
> A simple, focused solution that clearly changes user behavior in a meaningful way.
